🎯 HOW IT WORKS IN 3 STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: APPROVAL
  Receptionist clicks "Approve" in Appointment Management
  ↓
  HTTP POST /api/appointments/{id}/approve

STEP 2: BACKEND PROCESSING
  Backend updates database and emits WebSocket event
  ↓
  socketio.emit('appointment_approved', {queue_item_data})

STEP 3: INSTANT UPDATE
  All connected clients receive the event and update UI
  ↓
  User sees new patient in queue with animation! ✨

TOTAL TIME: < 100ms (Web Socket)
           5 seconds (HTTP Polling fallback)

Common Issues & Solutions:

Q: Queue doesn't update in real-time?
A: 1. Check backend is running (python3 app.py)
   2. Verify VITE_API_URL is set correctly
   3. Check browser console for errors (F12)
   4. See REAL_TIME_QUICK_START.md for details

Q: "Live Updates" shows disconnected?
A: 1. Verify backend port 5000 is accessible
   2. Check firewall/proxy settings
   3. Look at backend logs for errors
   4. Try refreshing the page

Q: Toast notifications not appearing?
A: 1. Verify socket event is emitted
   2. Check socket listener is attached
   3. Verify sonner package is installed
   4. Check browser console for JS errors

See REAL_TIME_QUICK_START.md for complete troubleshooting guide!
